# Основные встроенные функции

_ПРИМЕЧАНИЕ_: Все встроенные функции, перечисленные в этом документе,
неявно объявляются в глобальной области видимости до любого
пользовательского кода. Они обрабатываются как обычные вызовы функций
синтаксическим анализатором и проверяются на типы как встроенные модули
компилятора (intrinsics) во время семантического анализа.
Их имена зарезервированы глобально и не могут быть переопределены
пользователем.
см.
## 1. Ввод/Вывод и взаимодействие с файлами

(_Сюда ссылаются_: `semantics.md` параграф 8
"Автомат состояний Ввода/Вывода" пункт 2)

- **Контроль целей (Перенаправление)** _(см. `semantics.md` параграф 8_
  _"Автомат состояний Ввода/Вывода")_
  1. `функ нацелить_вывод: стр имя_файла -> !лог` _(Пытается открыть указанный
     файл в режиме перезаписи. В случае успеха, закрывает текущий целевой
     файл вывода (если это не `stdout`) и устанавливает новый файл как активный
     вывод. Возвращает `истина`. При сбое (неудаче) возвращает `ложь` и
     безопасно оставляет текущий активный вывод без изменений.)_
  2. `функ нацелить_вывод_доп: стр имя_файла -> !лог` _(Пытается открыть
     указанный файл в режиме добавления. В случае успеха, закрывает текущий
     целевой файл вывода (если это не `stdout`) и устанавливает новый файл как
     активный вывод. Возвращает `истина`. При сбое возвращает `ложь` и
     безопасно оставляет текущий активный вывод без изменений.)_
  3. `функ вернуть_вывод: -> ничто` _(Если текущим файлом вывода не является
     `stdout`, функция закрывает его и восстанавливает активный вывод обратно
     на `stdout`.)_
  4. `функ нацелить_ввод: стр имя_файла -> !лог` _(Пытается открыть указанный
     файл в режиме чтения. В случае успеха, закрывает текущий целевой файл
     ввода (если это не `stdin`) и устанавливает новый файл как активный
     ввод. Возвращает `истина`. При сбое возвращает `ложь` и безопасно
     оставляет текущий активный ввод без изменений.)_
  5. `функ вернуть_ввод: -> ничто` _(Если текущим файлом ввода не является
     `stdin`, функция закрывает его и восстанавливает активный ввод обратно
     на `stdin`.)_
  6. `функ нацелить_вывод_изм: стр имя_файла -> !лог` _(Пытается открыть
     **существующий** указанный файл в режиме чтения-записи без усечения
     (Транслируется в C++ как `std::ios::in | std::ios::out`). В случае
     успеха, закрывает текущий файл вывода (если это не `stdout`) и
     устанавливает новый файл как активный вывод. Возвращает `ложь`, если файл
     не существует или его не удалось открыть, безопасно оставляя текущий
     вывод активным. Возвращает `истина` при успехе.)_

- **Операции чтения (Из активной цели ввода)**
  1. `функ ввод: стр подсказка -> стр` _(Считывает строку до символа переноса
     строки. Если активной целью ввода является `stdin`, `подсказка`
     выводится в `stdout`.)_
  2. `функ читать_символы: ц32б кол_во -> !стр?` _(Считывает ровно указанное
     количество символов)._
  3. `функ читать_слова: ц32б кол_во -> !ряд[стр]?` _(Считывает массив из N слов,
     разделенных пробелами)._
  4. `функ задать_поз_ввода: ц32б байт -> ничто` _(Устанавливает курсор чтения
     в точно указанную абсолютную позицию байта)._
  5. `функ текущая_поз_ввода: -> ц32б` _(Возвращает текущую байтовую позицию
     курсора чтения)._
  6. `функ размер_ввода: -> ц32б` _(Возвращает суммарный размер активной цели
     ввода в байтах)._
  7. `функ конец_ввода: -> лог` _(Возвращает `истина`, если активная цель ввода
     достигла Конец Файла (EOF))._

- **Операции записи (В активную цель вывода)**
  1. `функ вывод: стр текст -> ничто` _(Отправляет текст в текущую активную
     цель вывода)._
  2. `функ вывод_ошибки: стр текст -> ничто` _(Отправляет текст в `stderr`)._
  3. `функ задать_поз_вывода: ц32б байт -> ничто` _(Устанавливает курсор записи
     в точно указанную абсолютную позицию байта)._
  4. `функ текущая_поз_вывода: -> ц32б` _(Возвращает текущую байтовую позицию
     курсора записи)._
  5. `функ размер_вывода: -> ц32б` _(Возвращает суммарный размер активной цели
     вывода в байтах)._

- **Усечение файлов (Не зависит от состояния)**
  1. `функ обрезать_файл: стр имя_файла, ц32б байтов_оставить -> !лог`
     _(Обрезает указанный файл мгновенно, оставляя только указанное количество
     байтов от начала. Возвращает булево)._

## 2. Управление исполнением и прочие утилиты:

- Завершение программы с кодом возврата: `функ выйти: ц32 -> ничто`
- Экстренный выход с сообщением об ошибке: `функ авария: стр -> ничто`
- Получение длины строки в символах UTF-8 (встроенная функция компилятора,
  статически типизируемая при использовании): `функ стр_длина: стр -> ц32б`
- Получение размера строки в байтах (встроенная функция компилятора, статически
  типизируемая при использовании): `функ стр_размер: стр -> ц32б`
- Получение длины динамического массива (встроенная функция компилятора,
  статически типизируемая при использовании):
  `функ ряд_длина: ряд[тип] -> ц32б`
- Получение длины статического массива: известно во время компиляции:
  `ст_ряд_длина(<expr>) -> ц32б` _(Обрабатывается компилятором как оператор,
  вычисляется в размерность предоставленного статического массива)_
- Добавление в конец динамического массива (встроенная функция компилятора,
  статически типизируемая при использовании):
  `функ пополн: изм ряд[тип], тип арг -> !лог` _(Добавляет одиночный элемент
  в конец массива. Возвращает `истина` при успехе.)_
  `функ пополн: изм ряд[тип], ряд[тип] арг -> !лог` _(Добавляет все элементы
  из массива `арг` в конец целевого массива. Возвращает `истина` при успехе.)_
- Удаление последних N элементов из динамического массива (встроенная функция
  компилятора, статически типизируемая при использовании):
  `функ удалить_посл: изм ряд[тип], ц32б кол_во -> ничто` *(Удаляет
  `кол_во` элементов с конца массива. Если `кол_во` превышает длину
  массива, возникает паника во время исполнения.)*
- _Вставка и удаление по произвольному индексу отложены до версий после MVP._
- Создание заполненного динамического массива (встроенная функция компилятора,
  статически типизируемая при использовании):
  `функ заполнить: тип знач, ц32б размер -> !ряд[тип]?` _(Возвращает новый,
  размещенный в куче динамический массив данного `размера`, каждый элемент
  которого инициализирован значением `знач`. Компилятор статически определяет
  `тип` из типа переменной `знач`.)_
- Получение размера в байтах для типов, известных во время компиляции:
  `размер(<type>) -> ц32б` _(Обрабатывается компилятором как оператор, а не
  как стандартная функция)_
- Обмен состояний (встроенная функция компилятора, статически типизируемая
  при использовании):
  `функ обмен: изм тип а, изм тип б -> ничто` _(Мгновенно меняет местами
  базовую память `а` и `б` без выполнения глубоких копий или выделений памяти
  в куче. `тип` является мета-плейсхолдером; оба аргумента должны строго
  определяться к идентичному типу. В бэкенде C++ это напрямую отображается
  на нативную функцию `std::swap(a, b)`)._

## 3. Операции со строками

(_Сюда ссылаются_: `semantics.md` параграф 4.А
"Преобразования типов (Приведение)")

Чтение текстовых данных и форматирование примитивов нативно. Эти встроенные
функции статически перегружаются компилятором для всех нативных стековых
примитивов (`цN`, `цNб`, `птN`, `лог`).

- **Конвертация примитива в строку:** `функ в_стр: <primitive> -> !стр?`
  _(Возвращает строковое представление значения. Возвращает ничто, если_
  _базовое выделение памяти в куче завершается неудачно)._
- **Конвертация строки в примитив:**
  `функ из_стр: стр текст, изм <primitive> итог -> !лог`
  _(Анализирует текст и сохраняет полученное преобразованное значение
  непосредственно внутри конкретно изменяемого экземпляра `итог`. Возвращает
  `истина` в случае успеха, или `ложь`, если строка имеет структурные
  дефекты, содержит недопустимые символы, или происходит переполнение числа.
  Для `лог` (логического типа), функция принимает строго строки `"истина"`
  и `"ложь"` (с учетом регистра). Иные значения возвращают `ложь`)._

_Примечание: `<primitive>` в сигнатурах выше — это вспомогательный_
_мета-плейсхолдер. Компилятор внутренне генерирует отдельную перегрузку для_
_каждого стекового примитивного типа (`ц8`..`ц64`, `ц8б`..`ц64б`, `пт32`,_
_`пт64`, `лог`). Пользователь вызывает эти функции как обычные, и компилятор_
_находит необходимую перегрузку статически из переданных типов аргументов._

- **Преобразование строки в массив UTF-32:**
  `функ в_ряд_символов: стр текст -> !ряд[ц32]?`
  (_Сюда ссылаются_: `semantics.md` параграф 5.Б
  "Семантика выражений и операторов" пункт "Семантика индексации строк UTF-8")

_(Возвращает совершенно новый динамический массив, содержащий кодовые позиции_
_UTF-32 (Руны) для этой строки. Это гарантирует время произвольного доступа_
_O(1) на символ, в обход задержек O(N) стандартного индексирования строки)._

- **Конвертация строки в массив байтов:**
  `функ в_ряд_байтов: стр текст -> !ряд[ц8]?`

_(Возвращает новый динамический массив, содержащий непосредственно сырые байты_
_UTF-8, лежащие в основе этой строки, без какого-либо перекодирования)._
