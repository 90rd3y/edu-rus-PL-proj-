# Грамматика языка (EBNF)

Этот документ определяет формальную лексическую и синтаксическую
грамматику языка. Используется нотация в БНФ, расширенная выражениями
группировки `()`, опциональности `[]` (0 или 1), повторения `{}` (0 или более).


## 1. Лексическая грамматика

```
<letter> ::= "А" | ... | "Я" | "а" | ... | "я" | "_"

<binary_digit> ::= "0" | "1"
<octal_digit> ::= "0" | ... | "7"
<digit> ::= "0" | ... | "9"
<hex_digit_except_zero> ::= "1" | ... | "9" | "А" | ... | "Е"
<hex_digit> ::= <digit> | "А" | ... | "Е"
@ Сопоставление кириллическим 16-ричным цифрам (буквам):
@ А=10, Б=11, В=12, Г=13, Д=14, Е=15

<id> ::= <letter> { <letter> | <digit> }
@ Ограничение: <id> должен отличаться от <keyword> и
@ имени встроенной функции (напр., ввод, вывод, заполнить).

<keyword> ::= "пер" | "пост" | "функ" | "структ" | "стат_структ" |
              "область" | "если" | "тогда" | "иначе" | "пока" | "вернуть" |
              "прекратить" | "продолжить" | "изм" |
              "включить" | "из" | "употреб" |
              "истина" | "ложь" | "ничто" |
              "и" | "или" | "не" |
              "ц8" | "ц16" | "ц32" | "ц64" |
              "ц8б" | "ц16б" | "ц32б" | "ц64б" |
              "пт32" | "пт64" | "лог" | "стр" |
              "ряд" | "стат_ряд" | "как"

<comment> ::= "@" { <any_char_except_newline> }

<dec_literal> ::= ("1" | ... | "9") { <digit> }
<hex_literal> ::= "0ш" <hex_digit_except_zero> { <hex_digit> }
<oct_literal> ::= "0в" ("1" | ... | "7") { <octal_digit> }
<bin_literal> ::= "0д1" { <binary_digit> }
<nonzero_int_literal> ::= <dec_literal> | <hex_literal> | <oct_literal> |
<bin_literal>
<int_literal> ::= "0" | <nonzero_int_literal>

<float_literal> ::= ("0" | <dec_literal>) "." <digit> {<digit>}
[ "э" [ "-" ] <dec_literal> ]
<bool_literal> ::= "истина" | "ложь"
<escape_seq> ::= "\" ( '"' | "\" | "н" | "о" | "0" )
<string_literal> ::= '"' { <any_char_except_quote_backslash_newline> |
<escape_seq> } '"'

<literal> ::= <int_literal> | <float_literal> | <bool_literal> |
<string_literal>

<spread_token> ::= "..."
```

@ --- Лексические ограничения и форматирование ---

@ Строковые экранируемые символы:
@ Следующие экранируемые символы являются единственными
распознаваемыми из таковых внутри <string_literal>:
@ \" — литерал двойной кавычки
@ \\ — литерал обратной косой черты
@ \н — новая строка
@ \о — пробельный отступ фиксированной ширины
@ \0 — нулевой байт
@ Любой другой символ после '\' внутри строкового литерала считается
@ ошибкой компиляции.

@ Ограничения на численные литералы:
@ Число является либо нулем, либо начинается не с него.
@ Двоичный, 8-ричный или 16-ричный нуль не имеет
@ отличного от `0` представления.

## 2. Синтаксическая грамматика

### 2.1 Структура программы

```
<program> ::= { <top_level_stmt> }
<top_level_stmt> ::= <declaration> | <module_stmt> | ";"
<statement> ::= <declaration> | <using_stmt> | <assignment_stmt> |
<control_flow> | <call_stmt> | <block_stmt> | ";"

<block_stmt> ::= "{" { <statement> } "}"
```

### 2.2 Модули, пространства имен и псевдонимы.

@ (_Сюда ссылаются_: `semantics.md` параграф 7
@ "Разрешение имен, области видимости и модули" пункты:
@ "Псевдонимы типов (Type Aliasing)", "Пространства имен (`область`)",
@ "Включение модулей")

```
<module_stmt> ::= <include_stmt> | <namespace_stmt> | <using_stmt>
<include_stmt> ::= "включить" <string_literal> ";" |
"из" <string_literal> "включить" <id> { "," <id> } ";"

<using_stmt> ::= "употреб" ( <type> <id> | <qualified_id> [ "\" "*" ] ) ";"

<namespace_stmt> ::= "область" <id> "{" <namespace_member>
{ <namespace_member> } "}"
<namespace_member> ::= <declaration> | <module_stmt>
```

### 2.3 Типы

@ (_Сюда ссылаются_: `types.md` параграф 1 "Синтаксис объявления типов",
@ `semantics.md` параграф 4.Б "Void, Nullability и двойственная природа `ничто`")

```
<primitive> ::= "ц8" | "ц16" | "ц32" | "ц64" | "ц8б" | "ц16б" | "ц32б" |
"ц64б" | "пт32" | "пт64" | "лог"

<type> ::= <primitive> | "стат_ряд" "[" <type> "," <nonzero_int_literal> "]" |
("стр" | <qualified_id> | "ряд" "[" <type> "]") [ "?" ]

@ <param_type>: variant of <type> for function parameters only.
@ Allows a user-named <id> as the стат_ряд size
@ (compile-time captured constant).
<param_type> ::= <primitive> |
"стат_ряд" "[" <param_type> "," (<nonzero_int_literal> | <id>) "]" |
("стр" | <qualified_id> | "ряд" "[" <param_type> "]") [ "?" ]
```

### 2.4 Объявления

@ (_Сюда ссылаются_: `types.md` параграф 1 "Синтаксис объявления типов",
@ `semantics.md` параграфы:
@ 1 "Модель исполнения" пункт
@ "Обязательные к проверке значения (Модификатор `!`)",
@ 2 "Память и присваивание"
@ пункт "Гарантия инициализации переменной")

```
<declaration> ::= <var_decl> | <func_decl> | <struct_decl>

<var_decl> ::= ("пер" | "пост") <id> ":" <type> "="
("ничто" | <spread_init> | <expr>) ";"

<struct_decl> ::= ("структ" | "стат_структ")
(<id> "{" <struct_field> { <struct_field> } "}" | <qualified_id> ";")
<struct_field> ::= <id> ":" <type> ";"

<func_decl> ::= "функ" (<id> ":" [ <param_list> ] "->" <ret_type> <block_stmt> |
<qualified_id> ":" [ <param_proto_list> ] "->" <ret_type> ";")
<ret_type> ::= [ "!" ] <type> | "ничто"
<param_proto_list> ::= <param_proto> {"," <param_proto>}
<param_proto> ::= [ "изм" ] <param_type> [ <id> ]
<param_list> ::= <param> { "," <param> }
<param> ::= [ "изм" ] <param_type> <id>
```

### 2.5 Инструкции и управление потоком исполнения

```
<control_flow> ::= <if_stmt> | <while_stmt> | <loop_control> | <return_stmt>

<loop_control> ::= ("прекратить" | "продолжить") ";"

<if_stmt> ::= "если" <expr> "тогда" <block_stmt>
[ "иначе" ( <block_stmt> | <if_stmt> ) ]
<while_stmt> ::= "пока" <expr> <block_stmt>

<return_stmt> ::= "вернуть" [ "ничто" | <expr> ] ";"

<lvalue> ::= <qualified_id> { "." <id> | "[" <expr> "]" }
<assignment> ::= <lvalue> "=" ("ничто" | <assignment> | <expr>) |
<lvalue> ("+=" | "-=" | "*=" | "/=") (<assignment> | <expr>)

<assignment_stmt> ::= <assignment> ";"
<call_stmt> ::= <qualified_id> "(" [ <arg_list> ] ")" ";"
```

### 2.6 Выражения

@ (_Сюда ссылаются_: `semantics.md`:
@ параграф 2 "Память и присваивание"
@ пункт "Вычисление оператора распространения (`...`)",
@ параграф 4.А "Преобразования типов (Приведение)" пункт "Явные приведения",
@ `types.md` параграф 6
@ "Синтаксис и правила преобразования типов (Приведение)")

```
<expr> ::= <logical_or>

<logical_or> ::= <logical_and> { "или" <logical_and> }
<logical_and> ::= <equality> { "и" <equality> }

<equality> ::= <relational> [ ("==" | "!=") ( <relational> | "ничто" ) ] |
"ничто" ("==" | "!=") <relational>
<relational> ::= <bitwise_or> [ ("<" | ">" | "<=" | ">=") <bitwise_or> ]

<bitwise_or> ::= <bitwise_xor> { "|" <bitwise_xor> }
<bitwise_xor> ::= <bitwise_and> { "^" <bitwise_and> }
<bitwise_and> ::= <bitwise_shift> { "&" <bitwise_shift> }
<bitwise_shift> ::= <term> { ("<<" | ">>") <term> }

<term> ::= <factor> { ("+" | "-") <factor> }
<factor> ::= <unary> { ("*" | "/" | "%") <unary> }

<unary> ::= ("-" | "~" | "не") <unary> | <primary>

<primary> ::= <literal> | <struct_init> | <array_init> |
(["как" "<" <type> ">"] "(" <expr> ")" | <qualified_id>) { <postfix> }

<postfix> ::= "(" [ <arg_list> ] ")" | "." <id> | "[" <expr> [ ":" <expr> ] "]"
<arg_list> ::= <arg> { "," <arg> }
<arg> ::= "ничто" | [ "изм" ] <expr>

<qualified_id> ::= <id> { "\\" <id> }
<struct_init> ::= <qualified_id> "{" ("ничто" | <expr> | <spread_init>)
{ "," ("ничто" | <expr> | <spread_init>) } "}"
<array_init> ::= "[" ("ничто" | <expr> | <spread_init>)
{ "," ("ничто" | <expr> | <spread_init>) } "]"
<spread_init> ::= <expr> <spread_token>
```

### 2.7 Приоритеты и ассоциативность операторов

@ (_Сюда ссылаются_: `semantics.md` параграф 5.Б
@ "Семантика выражений и операторов" пункт "Приоритет операторов")

@ Операторы вычисляются в порядке
@ от высшего приоритета (Уровень 1) к низшему (Уровень 13).
@ Операторы с одинаковым приоритетам вычисляются в
@ порядке согласно их ассоциативности.

- **Уровень 1** (Слева направо): `()`, `[]`, `.`, `\`
  - Группирование, индексация или срез строки, доступ к полю,
    разрешение области видимости / пространства имен.
- **Уровень 2** (Справа налево): `-`, `~`, `не`, `как<T>()`
  - Унарное отрицание, побитовая инверсия, логическое НЕ, приведение типа.
- **Уровень 3** (Слева направо): `*`, `/`, `%`
  - Умножение (Перегружено для `стр * N`), деление, остаток от деления.
- **Уровень 4** (Слева направо): `+`, `-`
  - Сложение (Перегружено для `стр + стр`), вычитание.
- **Уровень 5** (Слева направо): `<<`, `>>`
  - Побитовый сдвиг влево, вправо соответственно.
- **Уровень 6** (Слева направо): `&`
  - Конъюнкция (побитовое И)
- **Уровень 7** (Слева направо): `^`
  - Исключающее ИЛИ
- **Уровень 8** (Слева направо): `|`
  - Дизъюнкция (побитовое ИЛИ)
- **Уровень 9** (Неассоциативны): `<`, `<=`, `>`, `>=`
  - Операции отношения.
- **Уровень 10** (Неассоциативны): `==`, `!=`
  - Операции отношений равенства, неравенства соответственно.
- **Уровень 11** (Слева направо): `и`
  - Логическое И
- **Уровень 12** (Слева направо): `или`
  - Логическое ИЛИ
- **Уровень 13** (Справо налево): `=`, `+=`, `-=`, `*=`, `/=`
  - Присваивания (`+=`, `*=` перегружены для `стр`).
