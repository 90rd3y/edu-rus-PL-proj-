# C++ Development Rules

## Formatting
- **Strict Adherence:** All code generated must strictly follow the rules in the `.clang-format` file located in the project root.
- **Verification:** Before finalizing any code changes, mentally verify that the layout matches the project's formatting style.
- **No Variations:** Do not use personal style preferences; use the project's C++20 standard.

## Technical Context
- **Environment:** We are working in a WSL Ubuntu environment.
- **Project Goal:** This is a custom educational programming language project using C++20.
- **Git Hygiene:** Do not include massive reformatting in logic commits. If reformatting is needed, suggest it as a separate step.
